Security Policy Responsible Disclosure

Thanks for taking the time to help keep this site safe. If you believe you’ve found a security issue, please report it responsibly and I’ll do my best to respond quickly. Please hammer at this end point - if you can get in I'm shit.

How to Report

Email: security@bellis.tech (or stevie@bellis.tech)

Please include:

  • A clear description of the issue and impact
  • Steps to reproduce (URLs, parameters, payloads, screenshots/logs if relevant)
  • Your expected vs. actual result
  • Any suggested remediation (optional)

If you want to encrypt details, ask for a PGP key in your first message (or include yours).

Scope

This policy covers:

  • bellis.tech and its subpaths
  • Static site content and site infrastructure under my control

Out of scope:

  • Issues in third-party services not controlled by me
  • Denial of Service testing, automated scanning that disrupts service, or social engineering

Disclosure Guidelines

  • Please don’t access data that isn’t yours, modify data, or degrade availability.
  • Use the minimum testing needed to demonstrate the issue.
  • Give me a reasonable window to fix before public disclosure.

I’ll acknowledge receipt and aim to provide status updates as I investigate and remediate.

Acknowledgments

If you’d like public credit, tell me the name/handle and link you want listed here after the issue is resolved.
